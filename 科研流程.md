### 科研工作流程 (Research Workflow)

#### 概述
本技能为 AI 提供一套系统化的科研工作流程，涵盖从选题到成果发布的完整路径。适用于学术论文、技术报告、项目申请等研究任务。

> ⚠️ **免责声明**：本 skill 仅提供流程指导，实际研究需遵守所在机构、资助方及期刊的规范。AI 仅作为辅助工具，最终成果由用户负责。

---

##### 阶段一：选材（选题与材料收集）

**目标**：明确研究问题，收集背景资料和必要数据。

###### 1.1 选题
- 分析当前领域热点与空白
- 评估可行性：数据可获得性、技术成熟度、时间成本
- 形成清晰的研究问题（Research Question）

###### 1.2 文献调研
- 检索核心数据库（如 arXiv、Google Scholar、IEEE Xplore）
- 归纳已有工作的贡献与局限
- 确定本研究的创新点

###### 1.3 数据与材料准备
- 收集公开数据集或实验数据
- 准备代码仓库、工具链
- 整理参考文献（使用 Zotero、Mendeley 等）

**产出**：
- 研究计划书（含问题、创新点、可行性）
- 文献综述笔记
- 数据集清单 / 数据获取脚本

---

##### 阶段二：审核（内部或同行评审）

**目标**：在投入大量资源前，对研究设计进行批判性审视。

###### 2.1 设计评审
- 检查研究问题是否清晰、可验证
- 评估实验设计的科学性（对照组、样本量、统计方法）
- 确认数据与方法的合规性（伦理、隐私）

###### 2.2 可行性验证
- 小规模预实验或原型开发
- 估计所需算力、时间、人力
- 识别潜在风险（数据缺失、工具限制）

**产出**：
- 评审报告（含修改建议）
- 调整后的研究设计
- 风险评估与应对措施

---

##### 阶段三：编码（实现与实验）

**目标**：将设计转化为可运行的代码，执行实验并记录结果。

###### 3.1 环境搭建
- 配置开发环境（Python/conda、Docker 等）
- 版本控制（Git），建立清晰的 README
- 编写可复现的脚本（参数化、随机种子固定）

###### 3.2 核心实现
- 模块化开发，每个功能独立测试
- 遵循代码规范（PEP 8、类型注解）
- 添加详细注释和文档字符串

###### 3.3 实验执行
- 运行实验，记录每次运行的输出、配置、日志
- 使用实验管理工具（如 MLflow、Weights & Biases）
- 收集结果数据（指标、图表）

**产出**：
- 可运行代码仓库（含 README、requirements.txt）
- 实验日志与原始输出
- 初步结果可视化

---

##### 阶段四：写文（Markdown 格式论文/报告）

**目标**：以 **Markdown 格式**撰写研究论文或技术报告，保持轻量化、易版本控制，并可后续转换为 PDF/HTML 等正式格式。

###### 4.1 结构搭建
- 使用 Markdown 语法组织章节（`#`、`##`、`###`）
- 按标准学术结构：摘要、引言、方法、实验、结论、参考文献
- 可使用 `<!-- 注释 -->` 添加写作说明

###### 4.2 内容填充
- **摘要**：一句话概括问题、方法、结果、意义
- **引言**：背景、现有工作、本研究贡献
- **方法**：详细描述模型、算法、参数设置
- **实验**：数据集、评价指标、对比方法、结果分析
- **结论**：总结、局限、未来工作

###### 4.3 图表与引用
- 使用 `![alt text](path/to/image.png)` 插入图表
- 使用 Pandoc 式引用：`[@bibtex_key]`，并在文末列出参考文献列表
- 推荐使用 Zotero + Better BibTeX 导出 `.bib` 文件，用 `pandoc-citeproc` 自动生成参考文献

###### 4.4 可复现性说明
- 在附录或独立章节提供代码仓库链接、数据访问方式
- 说明如何运行实验复现结果

**产出**：
- **完整的 Markdown 论文草稿**（`.md` 文件）
- 图表文件（矢量图优先，存放于 `figures/` 文件夹）
- 参考文献 `.bib` 文件
- 可选：通过 Pandoc/Quarto 生成的 PDF/HTML 版本

> 💡 **推荐工具**：使用 `pandoc` 将 Markdown + BibTeX 转换为符合期刊要求的 PDF；或使用 `Quarto` 进行学术写作，支持多格式输出。

---

##### 阶段五：审核（同行反馈与修改）

**目标**：通过同行评审提升稿件质量。

###### 5.1 内部评审
- 邀请同事或导师阅读 Markdown 文件
- 收集反馈（逻辑漏洞、表达不清、图表问题）

###### 5.2 外部评审模拟
- 根据目标期刊/会议的审稿标准自查
- 检查创新性、实验充分性、写作质量

###### 5.3 修改
- 逐条回应反馈，在 Markdown 中标记修改
- 进行语言润色（语法、专业术语）
- 补充实验（如有必要）

**产出**：
- 修订版 Markdown 论文
- 回复信（针对审稿意见）
